# Contributing Guidelines

Thank you for contributing to the RIB UI Team Learning Repository! To ensure a smooth collaboration and high-quality codebase, please follow these guidelines.

## Changelog

We follow [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) to maintain a history of changes.

- **Log your changes**: Every significant change outside personal folders should be documented in the [CHANGELOG.md](./CHANGELOG.md)
- **Format**: Use the standard format (Added, Changed, Deprecated, Removed, Fixed, Security).

## Commit Messages

We follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) for our commit messages. This helps in generating changelogs and communicating the nature of changes.

## Merges

Please use Squash Merge and sum up your changes with a clear commit message.

**Examples:**

- `feat(angular): add new component examples`
- `fix(eslint): correct linting rule for unused vars`
- `docs: update readme with setup instructions`

**Common Types:**

- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation only changes
- `style`: Changes that do not affect the meaning of the code (white-space, formatting, etc)
- `refactor`: A code change that neither fixes a bug nor adds a feature
- `test`: Adding missing tests or correcting existing tests
- `chore`: Changes to the build process or auxiliary tools and libraries such as documentation generation

## Pull Requests

To ensure your code is reviewed quickly and effectively, please use the provided PR template.

**Your PR should include:**

1.  **Changelog**: A summary of what has changed.
2.  **UI Changes**: Screenshots or screenrecording of "Before" and "After" states (if applicable).
3.  **Test Cases**: How the reviewer can verify your changes.
4.  **Notes for Reviewer**: special setup, data requirements, or context.

When you open a PR, a template will automatically be loaded. Please fill out all relevant sections.
